# 当前架构功能说明

## Core

### GameBootstrap
- 启动项目。
- 切换 MainMenu / Run 场景。
- 统一接收输入按键。

### GameLoop
- 保存当前游戏状态。
- 驱动状态切换与更新。

### GameInput
- 获取玩家输入。
- 提供给核心流程使用。

### GameTimer
- 提供简单计时。

### GameStateId / IGameState
- 定义最小状态结构。

## Gameplay

### TypingSession
- 接收字符输入。
- 累积输入结果。
- 生成能量收益。

### TypingStats
- 统计总输入、正确数、连击等。

### ResourceWallet
- 保存能量和金币。
- 支持增加和扣除。

### BuildPrototype
- 保存塔的最小状态：名称、花费、等级。

### BuildService
- 消耗能量并提升塔等级。

### CombatEnemy
- 敌人从刷怪点向中心移动。
- 到达中心后销毁。

### CombatWaveManager
- 定时刷怪。
- 生成敌人对象。

### TopDownCameraRig
- 提供固定俯视角。

## UI

### RunHudController
- 显示当前能量和塔等级。
- 监听输入，推动打字产能。

### SceneMenuView
- 菜单按钮处理。
- 进入 Run 场景。

## 当前原则
- UI 只显示数据。
- Gameplay 只处理规则。
- Core 只做流程。
- Unity 场景尽量少绑定。
