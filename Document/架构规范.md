# 游戏项目架构规范

## 核心原则

游戏逻辑必须完全独立于 Unity。

Unity 只负责：

- 渲染
- 动画
- UI显示
- 输入收集
- 音效与特效

禁止让 Unity 承担游戏规则计算。

---

# 项目分层

## 1. 配置层（Config）

职责：

- 技能配置
- 敌人数值
- 武器参数
- 掉落概率

推荐：

- JSON
- ScriptableObject
- Excel导表

规则：

- 只存静态配置
- 不存运行状态

示例：

```json
{
  "bulletSpeed": 10,
  "damage": 20
}
```

---

## 2. 数据层（Data）

职责：

保存运行时状态。

例如：

- 玩家HP
- 金币
- 子弹数量
- Buff状态

规则：

- 只存数据
- 不处理逻辑
- 不处理显示

示例：

```csharp
class PlayerData
{
    public int hp;
    public int gold;
}
```

---

## 3. 逻辑层（Logic）

职责：

- 游戏规则
- 伤害计算
- 移动模拟
- 碰撞检测
- 敌人AI
- 技能系统
- 状态机

规则：

- 必须是纯C#
- 不允许依赖 Unity
- 不允许使用 MonoBehaviour
- 不允许使用 Transform
- 不允许使用 Rigidbody
- 不允许使用 Collider
- 不允许直接播放动画

允许：

- 数学计算
- 数据处理
- Update循环
- 状态判断

示例：

```csharp
class BulletLogic
{
    public float x;
    public float z;

    public float vx;
    public float vz;

    public void Update(float dt)
    {
        x += vx * dt;
        z += vz * dt;
    }
}
```

---

## 4. 显示层（View）

职责：

- 同步逻辑数据到Unity
- 更新模型位置
- 播放动画
- 播放特效
- 显示UI

规则：

- 不允许写游戏规则
- 不允许写伤害计算
- 不允许写AI逻辑
- 不允许修改核心数据

显示层只读取逻辑层状态。

示例：

```csharp
class BulletView : MonoBehaviour
{
    public BulletLogic logic;

    void Update()
    {
        transform.position =
            new Vector3(logic.x, 0, logic.z);
    }
}
```

---

# 数据流规则

允许的数据流：

```text
输入(Input)
    ↓
逻辑层(Logic)
    ↓
显示层(View)
```

禁止：

```text
View直接修改Logic
```

---

# UI规范

## UI职责

UI只负责显示数据。

允许：

- 血条刷新
- 文本刷新
- 按钮点击
- 图标显示

禁止：

- 伤害计算
- 技能逻辑
- 背包逻辑
- 战斗逻辑

---

## UI示例

```csharp
class HpBarView : MonoBehaviour
{
    public Slider slider;

    void Update()
    {
        slider.value =
            player.hp / player.maxHp;
    }
}
```

---

# 输入层规则

输入层只负责：

- 获取玩家操作
- 通知逻辑层

禁止：

- 在输入层计算游戏规则

示例：

```csharp
button.onClick.AddListener(() =>
{
    LogicWorld.UseSkill(1);
});
```

---

# 碰撞规则

禁止使用：

- Unity Physics
- OnCollisionEnter
- Rigidbody碰撞

使用：

- 自定义碰撞计算

示例：

```csharp
float dist =
    Vector2.Distance(a, b);

if(dist < radius)
{
    Hit();
}
```

---

# 子弹系统规范

逻辑层：

只处理：

- 坐标
- 速度
- 伤害
- 生命周期

显示层：

只处理：

- 模型位置
- 拖尾
- 粒子特效
- 动画

---

# AI生成代码规范

AI生成代码时必须遵守：

## 必须

- Logic与View分离
- Logic可脱离Unity运行
- View只读取Logic状态
- 使用纯数据结构
- 使用简单Update循环

## 禁止

- Logic继承MonoBehaviour
- Logic调用Transform
- Logic使用Unity Physics
- View写游戏规则
- 混合逻辑与显示

---

# 推荐架构方向

推荐：

- 数据驱动
- 组合优于继承
- 简单循环更新
- 模块化系统

避免：

- 深层继承
- 大量脚本互相引用
- Unity生命周期耦合
- 复杂组件依赖

---

# 最终目标

Unity只是显示器。

真正的游戏世界运行在：

```text
LogicWorld
```

所有游戏规则必须优先存在于逻辑层。
