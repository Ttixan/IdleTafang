# 开发计划（MVP）

## 1. 目标

交付一条可玩的闭环：**开局 Energy → 战斗中杀敌获能 → 波间消耗强化 / 建造 → 结算沉淀**，并能承载后续扩展（不做首发商业化）。

## 2. 原则

- **闭环优先**：先打通最小玩法骨架，再迭代手感与内容密度。
- **先骨架后打磨**：允许占位 UI 与简化数值；高风险模块尽早暴露接口。
- **逻辑与视图分离**：核心规则尽可能留在纯数据/服务层，Unity 场景只做组装。
- **不做首发商业化**：支付、账号、复杂投放与长线养成一律延后。
- **工程约束**：中文文案统一 UTF-8（含 BOM）；Unity 版本与现有工程保持一致。

## 3. 玩法要点（合并概述）

以下仅为当前实现的共识摘要；细节与接线以代码为准（`GameBootstrap`、`RunHudController`、`RunSession`、`CombatWaveManager`、`SectorFocus*`、`WaveIntermissionPanelView` 等）。

- **阶段**：`RunPhaseController` 单局默认从 **战斗** 开局（结算后进 Settlement）；已无打字准备阶段。
- **Energy**：`RunConfig.StartingEnergy` 开局注入钱包；杀敌按 **`CombatEnemy.EnemyTypeId`** × **当前波次**（`GetKillEnergyReward` × `waveKillEnergyMultipliers`）入账。
- **扇区主循环**：仅 **扇区自动炮**（`SectorFocusSystem` / `SectorFocusCombatAdapter`）；手无鼠标炮台。
- **波间**：战斗结束弹出强化面板（来源 `IntermissionBuffCatalog`），可选修理基地；强化状态进入 HUD 与结算摘要。
- **配置**：波次数、开局 Energy、杀敌能量表与波次倍率等由 **`RunConfig`**；Buff 池权重由 **`BuffPoolConfig`**。
- **本期不做**：番茄钟式准备档位分级（原 B）、特攻机制拆分与局外解锁树（原 D）、商业化。

## 4. 下一步（唯一进度来源）

按优先级从上到下执行；完成一项在 §7 **按自然日**追加当日条目（通常写在已有日期块内或在该日新建小节）。

1. **补齐 RunConfig / BuffPoolConfig 资产**：在工程中创建 ScriptableObject 资源并填入合理默认值；确认 `GameBootstrap` / `RunHudController` 引用无缺失。
2. **验收配置化路径**：改波次数、Buff 可选数量、权重表后，一局流程仍完整（波间 → 战斗 → 结算）。
3. **平衡与手感**：杀敌 Energy / Gold、`waveKillEnergyMultipliers`、扇区切换节奏与 Buff 体感（可调 `RunConfig`）。
4. **内容与占位抛光**：敌人/波次占位替换、结算面板可读性、HUD 信息层级。
5. **自动化**：关键逻辑测试继续保持绿灯；视需要补充 Buff 抽取与波次边界的用例。

### 验收清单（仍适用于 MVP）

- `Boot → MainMenu → Run → GameOver → Retry` 可重复跑通。
- 建造升级消耗正确反映；战斗胜负条件可用；结算金币持久化。
- 开局即有 Energy；战斗中杀敌入账；扇区自动炮与波间流程可用。
- 文档与本节「下一步」一致或无冲突。

### 风险与对策（摘要）

| 风险 | 对策 |
|---:|---|
| 手感不足导致「不像游戏」 | 先做数值旋钮（配置与 Inspector），再谈机制膨胀 |
| UI 返工 | MVP 允许占位；Presenter 与视图拆开 |
| 战斗复杂度溢出 | 先做少量敌人与清晰胜负；扇区与 Buff 增量迭代 |
| 杀敌经济难平衡 | 用 `RunConfig` 分类型 / 分波倍率旋钮迭代 |

## 6. 技术债与延后重构（精简）

以下为已知改进方向，**不阻塞 MVP 闭环**；实施时在 §7 记账。

- **波次与 Buff**：`CombatWaveManager` 与扇区/Buff 契约可进一步收紧（减少重复分支、统一失败路径）。
- **路径与移动**：`CombatPath` 仍为采样驱动；后续可抽象「槽位/路段」便于特效与机关。
- **对象生命周期**：敌人实例化策略偏 MVP；后续对象池与统一 Despawn。
- **输入与 UI**：扇区/建造/波间多条输入路径；后续收敛为单一调度或明确优先级表。
- **测试**：补充 BuffCatalog 边界、多波次与 Phase 切换的集成级用例。
