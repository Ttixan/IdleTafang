# 第二周与战斗资产接线

## 第二周要做什么

目标是把打字产能做成一个稳定、可见、可测试的最小闭环。

```text
Keyboard Input
  -> TypingInputRouter
  -> TypingSession
  -> TypingRewardCalculator
  -> ResourceWallet
  -> RunHudController / BuildPanelView
```

### 第二周产物
- 输入采集
- 有效字符判定
- 正确数、连击、准确率
- 能量奖励
- HUD 数值刷新
- 建造按钮能消费能量

### 第二周验证标准
- 输入任意字母、数字、标点后，能量会增加
- 空格和控制字符不会计入有效输入
- HUD 能看到当前能量和塔等级
- 建造按钮可以消耗能量并让塔升级

## 战斗纯逻辑如何和资产联系

现在的结构是两层：

```text
Pure Logic
  CombatWaveManagerLogic
  CombatEnemyLogic
  CombatPathLogic
  CombatArenaLogic

Unity Assets / Views
  CombatWaveManager
  CombatEnemy
  CombatPath
  CombatArena
  TopDownCameraRig
```

### 接线原则
- 纯逻辑只负责规则、状态、位置计算
- Unity 组件只负责读取场景对象、实例化 prefab、同步 Transform
- 场景里不要把规则写到 MonoBehaviour 里
- prefab 只承载“长什么样、挂哪些组件、引用哪些子物体”

### 当前连接方式
- `CombatWaveManager` 持有 `CombatWaveManagerLogic`
- `CombatEnemy` 持有 `CombatEnemyLogic`
- `CombatPath` 提供 `CombatPathLogic`
- `CombatArena` 提供 `CombatArenaLogic`
- 这些 MonoBehaviour 都是纯逻辑的壳

### 建议的运行链路
```text
Run Scene
  -> CombatArena / CombatPath / CombatWaveManager
  -> CombatWaveManagerLogic 决定何时刷怪
  -> Instantiate CombatEnemy prefab
  -> CombatEnemy 把自己的初始坐标交给 CombatEnemyLogic
  -> CombatEnemyLogic 每帧更新位置
  -> CombatEnemy MonoBehaviour 把逻辑坐标写回 Transform
```

## 需要补哪些 Prefab

### 必做 Prefab
1. `CombatEnemy.prefab`
- 一个敌人实体
- 挂 `CombatEnemy`
- 放一个可见模型或简单几何体
- 可选挂 `Animator`，但不参与规则

2. `CombatArena.prefab`
- 战场中心或目标点
- 挂 `CombatArena`
- 里面放一个 `CenterPoint` 子物体作为中心标记

3. `CombatPath.prefab`
- 路径容器
- 挂 `CombatPath`
- 下面放多个 `CombatSpawnPoint` 子物体
- 每个子物体表示一个刷怪点

4. `RunHud.prefab` 或 Run 场景内 HUD 根节点
- 挂 `RunHudController`
- 连接 `BuildPanelView`、`HudView`
- 负责显示状态和按钮

### 推荐但可后做
5. `TopDownCameraRig.prefab`
- 固定俯视相机
- 挂 `TopDownCameraRig`
- 只负责镜头跟随/角度

6. `CombatSpawnPoint` 子物体模板
- 不一定要单独做 prefab
- 也可以直接在 `CombatPath.prefab` 里放多个空物体
- 关键是位置和引用清晰

### 暂时不必做的 prefab
- 子弹 prefab
- 特效 prefab
- 多塔 prefab
- 多敌人 prefab
- 复杂地形 prefab

## 最小场景搭法

### Run 场景里保留
- `CombatArena`
- `CombatPath`
- `CombatWaveManager`
- `TopDownCameraRig`
- `RunHudController`
- 一个敌人 prefab 预制引用

### 建议 Hierarchy
```text
Run
├── CombatArena
│   └── CenterPoint
├── CombatPath
│   ├── SpawnPoint_0
│   ├── SpawnPoint_1
│   └── SpawnPoint_2
├── CombatWaveManager
├── TopDownCameraRig
└── UI
    ├── Hud
    └── BuildPanel
```

## 你现在最该做的内容
- 先做 1 个敌人 prefab
- 先做 1 个战场中心 prefab
- 先做 1 个路径 prefab，带 3 个刷怪点
- 再把 Run 场景拖线接起来
- 最后让刷怪和移动跑起来

## 判断是否完成第二阶段
- 能从主菜单进入 Run
- HUD 能显示能量
- 输入能产能
- 建造能消耗能量
- 战斗能刷出敌人并朝中心移动
- 至少一波能跑完并能结算
