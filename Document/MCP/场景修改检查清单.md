# MCP 场景修改检查清单

## 目的
在使用 Unity MCP 修改场景时，避免重复创建对象、重复添加组件、以及对错误对象做修改。

## 每次修改前先检查
1. 先 `scene-list-opened`，确认当前打开的是正确场景。
2. 再 `gameobject-find`，按名称或路径确认目标对象是否已经存在。
3. 如果对象已存在，不要再 `gameobject-create`。
4. 如果目标对象已存在组件，先 `gameobject-find` 或 `gameobject-component-get` 检查组件列表。
5. 如果组件已存在，不要重复 `gameobject-component-add`。
6. 如果多个同名对象可能存在，优先按完整路径查找。
7. 如果要批量修改，先确认每个对象的引用是否稳定，再执行修改。
8. 修改后再次查询一次，确认没有重复对象或重复组件。

## 常见重复问题
- `CombatPath` 在同一个对象上重复挂载。
- `CombatSpawnPoint` 在同一个刷怪点上重复挂载。
- `CombatArena`、`TopDownCameraRig`、`CombatWaveManager` 被加到错误对象上。
- 刷怪点列表里出现重复引用。

## 推荐流程
1. 查询场景。
2. 查询对象。
3. 查询组件。
4. 创建缺失对象。
5. 挂缺失组件。
6. 绑定引用。
7. 再次验证。

## 约定
以后涉及场景对象操作时，优先遵守“先查再改、已存在则跳过”的原则。
