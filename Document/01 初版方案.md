# 塔防游戏初版方案（V0.1）

## 1. 项目定位

### 1.1 产品定位
- 方向：娱乐优先，兼顾轻度习惯激励。
- 核心体验：玩家在工作阶段通过打字积攒战斗资源，在休息阶段进行短平快塔防战斗，形成“输入行为 -> 战斗反馈 -> 成长回报”的正循环。

### 1.2 目标用户
- 核心用户：泛用户（不强绑定学习/办公场景）。
- 使用场景：碎片化游玩、桌面挂机、轻度专注伴随。

### 1.3 视觉与叙事方向
- 美术风格：像素简约。
- 世界观建议（MVP级别）：玩家经营一座边境据点，白天收集能量、夜晚抵御怪潮。

## 2. 版本目标与范围

### 2.1 首版目标（1-2个月）
- 做出“可持续玩1-2小时”的核心循环。
- 完成基础战斗闭环：打字产能 -> 造塔 -> 防守 -> 奖励 -> 局外成长。
- 验证3类防御塔的策略差异是否足够。

### 2.2 MVP范围（必须做）
- 计时与阶段切换（工作/休息）。
- 打字转能量的组合规则。
- 3类防御塔与基础升级。
- 4种敌人（普通/快速/高甲/精英）。
- 4类资源（能量、金币、材料、科技点）。
- 1条局外成长线（数值+解锁混合）。
- 基础结算与存档。

### 2.3 MVP暂不做（可延后）
- 排行榜与强社交系统。
- 复杂文明进化链与奇观系统。
- 大量剧情演出。

## 3. 核心玩法循环

### 3.1 单局循环（Core Loop）
1. 进入工作阶段（默认25分钟）。
2. 玩家打字获取能量与少量金币。
3. 进入休息战斗阶段（建议3-5分钟）。
4. 使用资源建造/升级防御塔，抵御怪物波次。
5. 战斗结束获得金币、材料、科技点。
6. 用奖励进行局外成长或解锁，开始下一轮。

### 3.2 循环层级
- 短期（秒到分钟）：打字连击、能量增长、即时反馈。
- 中期（10-30分钟）：多波战斗、塔配置调整、资源分配。
- 长期（小时到周）：解锁新塔/新敌人/新规则，逐步提高策略深度。

## 4. 阶段机制设计

### 4.1 工作阶段（25分钟）
- 目标：稳定产出能量，给休息战斗做准备。
- 输入行为：玩家在任意文本输入场景中触发计数（或内置输入框）。
- 反馈：
	- UI实时显示当前能量、连击、效率评分。
	- 每30秒进行一次节律判定并给出轻提示（优秀/稳定/波动）。

### 4.2 休息战斗阶段（3-5分钟）
- 目标：合理使用工作阶段积累的资源完成防守。
- 玩法：建塔、升塔、临场微调（如卖塔返还部分资源）。
- 失败策略：首版采用“仅有压力无硬失败”。
	- 即使防守失利也可进入下一轮，但奖励打折。
	- 目的：保持轻松节奏，降低挫败和流失。

## 5. 打字转资源规则（组合机制）

### 5.1 基础得分
- 有效按键：每次+1基础输入值。
- 有效字符：字母/数字/中文均计入（可配置）。

### 5.2 组合加成
- 准确率加成：
	- >= 98%：能量x1.3
	- 95%-97%：能量x1.15
	- < 95%：无加成
- 节律连击加成：
	- 连续稳定输入30秒后进入连击，每级+5%能量，最高5级。
	- 中断超过3秒降1级。
- 专注窗口奖励：
	- 每5分钟结算一次“专注片段”，达标额外给金币或材料。

### 5.3 产出映射（建议初值）
- 能量：主要由打字行为产出，用于建造/升级。
- 金币：小额持续产出，战斗结算为主来源。
- 材料：战斗结算和事件奖励产出，用于中级升级。
- 科技点：精英波次或阶段目标达成产出，用于局外解锁。

## 6. 防御塔与敌人设计

### 6.1 三类防御塔（首版）
- 速射塔（单体DPS）：
	- 优势：对普通怪效率高。
	- 劣势：对高甲单位吃力。
- 溅射塔（范围清杂）：
	- 优势：处理中小怪群。
	- 劣势：攻速慢，单体爆发低。
- 减速/功能塔（控制辅助）：
	- 优势：延长输出时间，提高整体防线效率。
	- 劣势：单独摆放收益低，依赖搭配。

### 6.2 敌人类型（首版）
- 普通怪：标准移动与血量。
- 快速怪：移速高、血量低，考验即时火力覆盖。
- 高甲怪：移速低、减伤高，需要功能塔配合。
- 精英怪：属性更高并带小技能，提供科技点奖励。

## 7. 资源与经济系统

### 7.1 资源定义
- 能量：局内即时资源，建造和即时升级核心消耗。
- 金币：局内主货币，塔建造与基础升级消耗。
- 材料：局内稀缺资源，中级升级或关键建筑消耗。
- 科技点：局外资源，用于永久成长和内容解锁。

### 7.2 消耗优先级
1. 基础建塔（能量+金币）
2. 关键升级（金币+材料）
3. 局外解锁（科技点）

### 7.3 防滚雪球策略
- 同类塔递增成本（第N座成本增长10%-15%）。
- 每波怪物强度平滑递增，避免突然数值墙。
- 结算最低保底奖励，确保弱局也有成长。

## 8. 局外成长（混合型）

### 8.1 数值成长
- 初始能量上限提高。
- 塔基础攻击/攻速小幅提高。
- 专注片段奖励系数提高。

### 8.2 内容解锁
- 解锁塔分支升级（例如速射塔穿透弹）。
- 解锁敌人变体和新地图机制。
- 解锁被动插件（工作阶段特殊加成）。

### 8.3 成长节奏
- 前30分钟快速解锁（高反馈）。
- 30-90分钟进入稳定成长（低幅但持续）。
- 避免硬卡点，优先保证“始终有可做决策”。

## 9. 奖励与反馈设计

### 9.1 即时反馈（打字中）
- 键入特效、连击提示、效率条。
- 达到阶段阈值时给出轻量音效和弹字反馈。

### 9.2 战斗反馈
- 敌人受击、击杀、漏怪提示。
- 每波结束展示伤害贡献和资源收益。

### 9.3 结算反馈
- 本轮效率评分（输入效率、战斗效率、资源管理）。
- 下一轮建议（如“建议补功能塔”）。

## 10. 风险与应对

### 10.1 主要风险
- 风险1：打字与战斗割裂，像两个独立小游戏。
- 风险2：3类塔策略深度不够，1小时后疲劳。
- 风险3：无硬失败可能导致挑战不足。

### 10.2 应对策略
- 用资源映射与阶段目标强化两端因果关系。
- 每类塔至少设计1条升级分支，形成搭配决策。
- 引入“奖励折损”而非“游戏结束”，平衡轻松与压力。

## 11. 数据与验证指标（首版）

### 11.1 体验指标
- 单次游玩时长目标：60-120分钟。
- 第1小时留存体验目标：玩家能完成至少4轮循环。

### 11.2 系统指标
- 工作阶段平均有效输入率。
- 每轮资源盈余/赤字占比。
- 3类塔使用占比（避免单一塔垄断）。

### 11.3 平衡指标
- 战斗奖励折损触发率（建议20%-40%）。
- 精英波次通过率（建议40%-70%）。

## 12. 开发里程碑（1-2个月）

### 第1-2周：核心骨架
- 计时器、阶段切换、基础输入统计。
- 简单地图、路径、刷怪与结算。

### 第3-4周：战斗可玩
- 3类塔、4类敌人、基础升级与资源系统。
- 打字组合机制接入并完成首轮平衡。

### 第5-6周：局外成长与打磨
- 科技点、解锁树、保底奖励和引导。
- UI与反馈完善、存档、基础数据埋点。

### 第7-8周（可选）：体验冲刺
- 数值二次平衡。
- 新手流程、说明文案、首版发布准备。

## 13. 待确认问题（下次迭代优先）
- 长期目标最终选型：排行榜 / 奇观建造 / 文明进化链 / 混合。
- 工作阶段是否需要“真实外部输入接入”还是只保留游戏内输入框。
- 单次休息战斗时长最终定为3分钟还是5分钟。
- 局外成长中数值与解锁的权重比例（建议先6:4）。
