# 归档：塔防游戏初版方案（V0.1）

> 状态：**已归档**。本文保留早期设想的完整草稿，用于回顾与对照；当前实现与计划以 `01_开发计划.md` 为准。

## 1. 项目定位

### 1.1 产品定位
- 方向：娱乐优先，兼顾轻度习惯激励。
- 核心体验：玩家在工作阶段通过打字积攒战斗资源，在休息阶段进行短平快塔防战斗，形成“输入行为 -> 战斗反馈 -> 成长回报”的正循环。

### 1.2 目标用户
- 核心用户：泛用户（不强绑定学习/办公场景）。
- 使用场景：碎片化游玩、桌面挂机、轻度专注伴随。

### 1.3 视觉与叙事方向
- 美术风格：像素简约。
- 世界观建议（MVP级别）：玩家经营一座边境据点，白天收集能量、夜晚抵御怪潮。

## 2. 版本目标与范围

### 2.1 首版目标（1-2个月）
- 做出“可持续玩1-2小时”的核心循环。
- 完成基础战斗闭环：打字产能 -> 造塔 -> 防守 -> 奖励 -> 局外成长。
- 验证3类防御塔的策略差异是否足够。

### 2.2 MVP范围（必须做）
- 计时与阶段切换（工作/休息）。
- 打字转能量的组合规则。
- 3类防御塔与基础升级。
- 4种敌人（普通/快速/高甲/精英）。
- 4类资源（能量、金币、材料、科技点）。
- 1条局外成长线（数值+解锁混合）。
- 基础结算与存档。

### 2.3 MVP暂不做（可延后）
- 排行榜与强社交系统。
- 复杂文明进化链与奇观系统。
- 大量剧情演出。

## 3. 核心玩法循环

### 3.1 单局循环（Core Loop）
1. 进入工作阶段（默认25分钟）。
2. 玩家打字获取能量与少量金币。
3. 进入休息战斗阶段（建议3-5分钟）。
4. 使用资源建造/升级防御塔，抵御怪物波次。
5. 战斗结束获得金币、材料、科技点。
6. 用奖励进行局外成长或解锁，开始下一轮。

### 3.2 循环层级
- 短期（秒到分钟）：打字连击、能量增长、即时反馈。
- 中期（10-30分钟）：多波战斗、塔配置调整、资源分配。
- 长期（小时到周）：解锁新塔/新敌人/新规则，逐步提高策略深度。

## 4. 阶段机制设计

### 4.1 工作阶段（25分钟）
- 目标：稳定产出能量，给休息战斗做准备。
- 输入行为：玩家在任意文本输入场景中触发计数（或内置输入框）。
- 反馈：
	- UI实时显示当前能量、连击、效率评分。
	- 每30秒进行一次节律判定并给出轻提示（优秀/稳定/波动）。

### 4.2 休息战斗阶段（3-5分钟）
- 目标：合理使用工作阶段积累的资源完成防守。
- 玩法：建塔、升塔、临场微调（如卖塔返还部分资源）。
